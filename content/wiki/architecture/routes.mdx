## Auth posture flowchart

```
visitor → /
  │
  ├── signed in   → redirect /home
  └── anonymous   → render landing

visitor → /(app)/*   (any auth-required route)
  │
  ├── signed in   → render
  └── anonymous   → redirect /login?next=<path>

visitor → /wiki/*
  │
  ├── admin email   → render
  ├── signed in     → redirect /home (silent — doesn't leak that /wiki exists)
  └── anonymous     → redirect /login?next=/wiki
```
