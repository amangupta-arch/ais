```
bundles                       — themed grouping; carries plan_tier + tags
  └─ courses                  — a coherent unit (e.g. "HCF and LCM")
       └─ lessons             — a single ~10-minute session
            └─ lesson_turns   — the atomic unit (one MCQ, one bubble, one fill-in)
```

## Translations

Every level (`bundles`, `courses`, `lessons`, `lesson_turns`) has a `translations` JSONB column keyed by language code:

```json
{
  "en": { "title": "Prime Factorization", "subtitle": "..." },
  "hi": { "title": "अभाज्य गुणनखंडन", "subtitle": "..." }
}
```

The canonical row's top-level columns (`title`, `description`, etc.) on `bundles` and `courses` hold the **English** version. The `translations` jsonb holds **non-English** versions. Helpers in `lib/types.ts` — `bundleTitle(b, lang)`, `lessonTitle(l, lang)`, etc. — resolve fallbacks:

1. Try `translations[lang].title`.
2. Fall back to `translations.en.title`.
3. Fall back to the canonical column.
4. Fall back to the slug (defensive).

## Lesson turns — the atomic unit

A turn is one screen of activity in the lesson player. Each turn has:

- `turn_type` (text) — see [turn types](/wiki/authoring/turn-types) for the full list (`tutor_message`, `mcq`, `fill_in_the_blank`, `drag_to_reorder`, `tap_to_match`, `free_text`, `reflection`, `exercise`, `ai_conversation`, `media`, `svg_graphic`, `html_animation`, `checkpoint`)
- `content` (jsonb) — turn-type-specific schema
- `translations` (jsonb) — per-language overrides for content fields
- `order_index` (int) — sequence within the lesson
- `xp_reward` (int) — points awarded on completion

When the player advances through turns, `current_turn_index` is written to `user_lesson_progress` so resumes land on the right step.

## How content gets into the DB

1. A **bundle** is inserted directly via SQL or seed (`supabase/seed.sql`, migration files).
2. **Courses + lessons** are authored as YAML in `supabase/content/bundle-courses/*.yaml` and applied via `scripts/load-bundle-courses.ts` — see [adding content](/wiki/authoring/adding-content).
3. **Turn content** is generated by the `/yaml-generate` tool — Claude writes the YAML, the tool applies it to `lesson_turns`, and the audio pipeline generates ElevenLabs mp3s in the same run. See [yaml-generate flow](/wiki/authoring/yaml-generate-flow).

## Filtering

The two surfaces that browse content filter by different things:

- **`/learn`** — by `plan_tier`, broken into Free / Basic / Advanced sections. Ignores curriculum bundles.
- **`/student`** — by `tags @> ['curriculum', 'class:<X>']` plus optional `institute:<slug>`. Excludes any bundle tagged for an institute when the user has no institute, so K-12 never leaks into NMIMS BBA's class slot and vice versa.
