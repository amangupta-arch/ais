## Text fallback chain

For any displayed string (bundle title, lesson title, turn content), helpers in `lib/types.ts` walk:

1. `translations[<lang>]` for the requested language.
2. `translations.en` (canonical English).
3. The top-level column (`title`, `description` — equivalent to English).
4. The slug (defensive, never expected in production).

If a learner picks `hi` and the lesson has no Hindi translation, they see the English text — not a blank screen.

## Audio fallback chain

Per the [audio-mirrors-text-fallback decision](/wiki/decisions/audio-mirrors-text-fallback):

1. If translated text exists for the user's language → request audio in that language.
2. If we fell back to canonical English text → request English audio.
3. If neither manifest entry exists → browser `speechSynthesis` (last resort, sounds notably worse).

## Fonts

Non-Latin scripts need Noto fallback fonts. Three families are loaded via Google Fonts in `app/layout.tsx`:

- Noto Sans Devanagari (Hindi, Marathi)
- Noto Sans Tamil
- Noto Sans Bengali

Punjabi (Gurmukhi) and Telugu rely on system fonts today — most Indian phones have them installed. Add to layout if rendering misbehaves.

## Adding a new language

1. Add an entry to `LANGUAGES` in `lib/types.ts`.
2. Add a voice ID in `lib/audio/voices.ts:VOICE_IDS` (null is fine; means audio skips for that language).
3. Pick a model in `MODEL_BY_LANGUAGE` (defaults to `eleven_multilingual_v2`).
4. Add the script-specific font in `app/layout.tsx` if non-Latin and not already loaded.
5. Update the `WIKI_ADMINS` regex on `profiles.preferred_language` if there's a CHECK (there isn't today; we trust the dropdown).
