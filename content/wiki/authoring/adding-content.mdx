The full process to add a new chapter or curriculum bundle to the catalogue.

## 1. Decide the slugs

Bundle slugs are unique and stable — pick carefully. Examples:

- `b-class-9-math-ch01-number-systems`
- `b-nmims-bba-sem02-marketing-foundations`

Course slugs are unique globally (not per-bundle). Lesson slugs are unique per-course. They're auto-generated by the loader from titles via `slugify()`.

## 2. Insert the bundle row (SQL or migration)

```sql
insert into bundles (slug, plan_tier, emoji, cover_gradient, order_index, tags, translations)
values (
  'b-class-9-math-ch01-number-systems',
  'student',
  '🔢',
  'paper',
  410,
  '{curriculum,class:9,subject:mathematics}',
  '{"en":{"title":"Class 9 Math · Ch.1 Number Systems","description":"..."}}'::jsonb
)
on conflict (slug) do nothing;
```

Apply via the Supabase MCP `apply_migration` tool or by pasting into SQL Editor.

## 3. Author the YAML

Create `supabase/content/bundle-courses/NN-<bundle-slug>.yaml`:

```yaml
bundles:
  - bundle_slug: b-class-9-math-ch01-number-systems
    courses:
      - title: "Real Numbers Refresh"
        outcome: "Student can place any number on the real line"
        lessons:
          - "What are Real Numbers"
          - "Rational vs Irrational"
          - "The Number Line"

      - title: "Surds and Their Properties"
        outcome: "Student can manipulate surds"
        lessons:
          - "Introducing Surds"
          - "Rationalising the Denominator"
```

The loader (`scripts/load-bundle-courses.ts`) slugifies titles to course/lesson slugs. Verify the resulting slugs are what you want — `slugify` lowercases, strips apostrophes, replaces non-alphanumeric runs with `-`, and truncates at 64 chars.

## 4. Run the loader

```sh
npx tsx scripts/load-bundle-courses.ts
```

Output goes to `/tmp/bundles/<bundle-slug>.sql`. Inspect, then apply via Supabase MCP `execute_sql` (or SQL Editor) — the SQL is idempotent (`ON CONFLICT DO UPDATE`).

## 5. Generate turn content (per lesson, per language)

Open `/yaml-generate`, pick the bundle → course → lesson → language → click Generate. The route:

1. Calls Claude to generate the lesson YAML.
2. Applies it to `lesson_turns`.
3. Runs the audio pipeline (ElevenLabs TTS → Supabase Storage → manifest).

Repeat for every `(lesson, language)` you want shipped.

## 6. Verify

- Visit `/learn/<courseSlug>/<lessonSlug>` — turns render in the player.
- For curriculum bundles, visit `/student` while signed in as a learner whose `school_class` matches — the bundle should appear under its subject.
- On `/yaml-status`, recent jobs show their outcome.

## What NOT to do

- Don't insert into `courses` or `lessons` by hand if you can avoid it — let the loader handle it. The loader writes `lesson_count`, sets sensible defaults, and ensures FKs are valid.
- Don't tag a non-school bundle with `curriculum` — it'll show up on `/student`.
- Don't run `/yaml-generate` for a language whose voice ID is null in `lib/audio/voices.ts` unless you've decided to ship without audio for that language; the pipeline will skip with `audio:skipped reason:"voice_id_missing"` (the YAML still applies fine).
